# Skill: sales-report
**Version:** 1.0
**Business:** Car Markaz
**Last Updated:** 2026-03-26

---

## Purpose
Generate daily or weekly sales reports for Car Markaz. Sheikh Ali Kabir drops raw data, and the AI produces a formatted business report.

## Trigger Conditions
Task file contains these keywords: `report`, `sales`, `daily`, `weekly`, `farokht`, `summary`, `performance`

---

## Step-by-Step Instructions

**Step 1: Identify Report Period**
Determine from the task file:
- Daily Report (one day) or Weekly Report (7 days)?
- Date range: [start date] to [end date]

**Step 2: Parse Input Data**
Extract the following data from the task file:

```
SALES:
- Vehicles sold: [count]
- Vehicle details: [model, price each]
- Total revenue: PKR [amount]

INQUIRIES:
- New inquiries received: [count]
- Source breakdown: WhatsApp [x] / Phone [y] / Walk-in [z] / Online [w]

ACTIVITIES:
- Follow-ups completed: [count]
- Test drives conducted: [count]
- Showroom visits: [count]

PIPELINE:
- Pending deals (almost finalized): [count + brief]
- Active leads: [count]
- Cold leads: [count]
```

**Step 3: Calculate Metrics**
- Total Revenue = sum of all sold vehicles
- Average Selling Price = Total Revenue / Vehicles Sold
- Inquiry-to-Visit Rate = (Showroom Visits / Total Inquiries) × 100
- Visit-to-Sale Rate = (Vehicles Sold / Showroom Visits) × 100
- Most inquired vehicle type

**Step 4: Compare Against Previous Report (If Available)**
- Search `Done/` folder for previous period report
- If found → compare:
  - Revenue change: +/- PKR [amount] ([percentage]%)
  - Inquiry change: +/- [count]
  - Change of 30% or more → flag as ALERT

**Step 5: Generate Recommendations**
Suggestions based on data:
- If a vehicle has high interest → "Need to add more stock"
- If inquiries are low → "Marketing boost needed"
- If many pending deals → "Should focus on follow-ups"

**Step 6: Build the Report**

Full Report format:

**Step 7: Save Output**
- Full report: `Plans/Reports/REPORT_[period]_[date].md`
- Executive summary only: `Pending_Approval/REPORT_SUMMARY_[date].md`

**Step 8: Cleanup**
- Move original task file to `Done/`
- Update Dashboard.md
- Write log entry

## Output Destinations
- Full report: `Plans/Reports/REPORT_[DAILY/WEEKLY]_YYYY-MM-DD.md`

## Escalation Conditions
- Revenue down more than 50% → Needs_Action URGENT
- Any fraud or unusual transaction → Needs_Action URGENT
- Data incomplete or inconsistent → Needs_Action for clarification
